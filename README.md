# Pestimator SaaS

Pestimator is a Next.js SaaS workspace for pest control companies to manage clients, build polished quotes, send secure proposal links, convert accepted work to invoices, and collect payment through Stripe.

## Current App

- `app/` and `src/` contain the production Next.js application.
- `amplify/` defines the Amplify Gen 2 backend for Cognito, DynamoDB-backed Data models, and S3 attachment storage.
- `public/` contains the previous static demo bundle and is no longer the production deployment target.

## Cloud Services

Production is designed for AWS Amplify Gen 2 SSR:

- Cognito email sign-in, self-serve signup, and TOTP MFA.
- DynamoDB persistence through the server-side store with `PERSISTENCE_DRIVER=dynamo`.
- S3 presigned uploads for quote attachments.
- SES quote and invoice delivery with webhook event ingestion.
- Stripe hosted invoice links and paid-status webhook sync.

## Local Development

```bash
pnpm install
pnpm dev
```

Local development defaults to file persistence at `.local/pestimator-dev.json`.

## Production Configuration

Set the values from `.env.example` in Amplify Hosting. The SSR build writes the required runtime values to `.env.production` in `amplify.yml`.

Required for production:

- `APP_URL`
- `SESSION_SECRET`
- `AWS_REGION`
- `COGNITO_CLIENT_ID`
- `COGNITO_USER_POOL_ID`
- `SES_FROM_EMAIL`
- `S3_ATTACHMENTS_BUCKET`
- `STRIPE_SECRET_KEY`
- `STRIPE_WEBHOOK_SECRET`

The app reads DynamoDB table names from `TABLE_*` environment variables or from `amplify_outputs.json` custom `dynamodbTables` generated by `amplify/backend.ts`.

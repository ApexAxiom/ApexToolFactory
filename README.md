# Pestimator SaaS

Pestimator is a Next.js SaaS workspace for pest control companies: manage clients, build polished quotes, send secure proposal links, schedule the resulting work on a service calendar, convert accepted work to invoices, and collect payment through Stripe.

## Current App

- `app/` and `src/` contain the production Next.js application.
- `amplify/` defines the Amplify Gen 2 backend for Cognito, DynamoDB-backed Data models, and S3 attachment storage.

## Core Workflows

- **Clients** — accounts, billing contacts, service properties, edit-in-place, full quote/visit history.
- **Quotes** — pricing engine (sqft rates, visit cadences, pest findings, markup, tax), PDF rendering, SES delivery, secure no-login customer portal with accept/decline capture.
- **Schedule** — month calendar with technician assignment. Accepted quotes automatically become jobs (one-time) or active service plans (monthly/quarterly) whose future visits are generated on a rolling 90-day horizon. Jobs flow through `UNSCHEDULED → SCHEDULED → IN_PROGRESS → COMPLETED` with a service report (work performed, materials/chemicals applied).
- **Invoices** — one-click conversion from accepted quotes, Stripe hosted payment links, webhook-driven paid status, manual payment recording for cash/check.
- **Team & access** — role-based permissions per membership (OWNER, OFFICE_MANAGER, ESTIMATOR, TECHNICIAN, ACCOUNTING) enforced in every server action.

## Security Posture

- All `/app` routes are protected by `middleware.ts` (JWT verification) plus per-request session and tenant checks.
- `SESSION_SECRET` (32+ chars) is mandatory at production runtime; the local dev login bypass is hard-disabled in production.
- Stripe webhooks require a valid signature and are deduplicated by event id; SES/SNS webhooks verify the SNS message signature before ingestion.
- Multi-record mutations (quote + revision + lines, invoice + lines, payment + balance) are written transactionally.
- Every server action validates its input with zod before touching data.

## Local Development

```bash
pnpm install
pnpm dev
```

Local development defaults to file persistence at `.local/pestimator-dev.json`.

## Production Configuration

Set the values from `.env.example` in Amplify Hosting. The SSR build writes the required runtime values to `.env.production` in `amplify.yml`.

Required for production:

- `APP_URL`
- `SESSION_SECRET`
- `AWS_REGION`
- `COGNITO_CLIENT_ID`
- `COGNITO_USER_POOL_ID`
- `SES_FROM_EMAIL`
- `S3_ATTACHMENTS_BUCKET`
- `STRIPE_SECRET_KEY`
- `STRIPE_WEBHOOK_SECRET`

The app reads DynamoDB table names from `TABLE_*` environment variables or from `amplify_outputs.json` custom `dynamodbTables` generated by `amplify/backend.ts`. Deploying the updated Amplify backend provisions the new `Job`, `ServicePlan`, and `WebhookEvent` tables.
